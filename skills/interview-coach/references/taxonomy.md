# 前端面试题分类

每道题选择一个主分类，可添加多个标签。优先按被考察的核心机制分类，而不是按题目中偶然出现的技术名词分类。

| 主分类 | 典型范围 |
| --- | --- |
| HTML 与 Web 标准 | 语义化、可访问性、SEO、DOM 标准、Web Components |
| CSS 与视觉布局 | 盒模型、布局、层叠、响应式、动画、渲染表现 |
| JavaScript | 类型、作用域、原型、异步、模块、语言机制 |
| TypeScript | 类型系统、泛型、推导、工程配置、类型设计 |
| 浏览器原理 | 事件循环、渲染、存储、跨域、缓存、进程模型 |
| 网络与协议 | HTTP、HTTPS、TCP、DNS、WebSocket、CDN |
| React | Hooks、调和、状态、并发、组件设计、生态 |
| Vue | 响应式、编译、组件通信、状态管理、生态 |
| Node.js | 运行时、事件循环、流、服务端、框架 |
| 前端工程化 | 构建、模块化、包管理、CI/CD、质量体系、微前端 |
| 性能优化 | 指标、测量、加载、运行时、渲染、监控 |
| 安全 | XSS、CSRF、CSP、认证授权、供应链安全 |
| 测试与质量 | 单元、集成、端到端、可测试性、故障定位 |
| 数据结构与算法 | 复杂度、常用结构、算法题、前端场景算法 |
| 系统设计与架构 | 组件/状态设计、稳定性、扩展性、可观测性、权衡 |
| 项目与行为面试 | 项目难点、协作、决策、复盘、影响力 |
| 其他 | 尚不能稳定归入以上分类的题目；后续应重新审视 |

拆分原则：一句话中若包含多个可以独立评分的核心问题，按用户原有顺序拆成多题；若只是同一机制的连续追问，保留为一道题并记录追问。
